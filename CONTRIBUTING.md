# Contributing

Thanks for contributing.

## Quick Start

1. Fork and clone the repository.
2. Create and activate a virtual environment.
3. Install dependencies:

```powershell
pip install -r requirements.txt
```

4. Run tests before opening a PR:

```powershell
pytest -q
```

## Pull Request Guidelines

- Keep changes focused and small.
- Add or update tests for behavior changes.
- Do not commit local artifacts (databases, reports, cache files).
- Use clear commit messages and PR descriptions.

## Code Style

- Follow existing project structure and naming style.
- Prefer readable, simple functions over large monolithic blocks.
- Keep UI text and documentation in English.
